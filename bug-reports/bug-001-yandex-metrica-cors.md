## Заголовок
При загрузке страницы master164.ru в консоль браузера выводятся ошибки CORS, связанные с загрузкой скриптов Яндекс.Метрики.

---

## Окружение (Environment)

| Компонент | Значение |
|-----------|----------|
| **Устройство** | MacBook Pro 16" 2021 |
| **ОС** | macOS Tahoe 26.2 |
| **Браузер** | Safari |
| **Сайт** | [https://master164.ru](https://master164.ru) |

---

## Шаги воспроизведения (Steps to Reproduce)

1. Открыть сайт [https://master164.ru](https://master164.ru) в браузере Safari.
2. Открыть инструменты разработчика (Веб-инспектор → вкладка **Консоль**).
3. Обновить страницу (`Cmd + R`).
4. Проанализировать сообщения в консоли.

---

## Фактический результат (Actual Result)

В консоли браузера отображается одна или несколько ошибок вида:
Fetch API cannot load https://mc.yandex.ru/webvisor/29956739?... due to access control checks.

- Ошибки связаны с загрузкой скриптов Яндекс.Метрики (вебвизор, счётчик).
- Ошибка повторяется при каждой загрузке страницы.
- Работа сайта **не блокируется**, но консоль сильно засорена.

---

## Ожидаемый результат (Expected Result)

- При загрузке сайта в консоли браузера **не должно быть ошибок**, связанных с загрузкой внешних скриптов.
- Счётчик Яндекс.Метрики должен загружаться и работать корректно, без конфликтов с CORS-политикой браузера.
- Консоль должна оставаться **чистой**, чтобы не затруднять отладку других скриптов и не вводить в заблуждение разработчиков и тестировщиков.

---

## Возможная причина

Скорее всего, на стороне сервера `master164.ru` некорректно настроены заголовки CORS для загрузки внешних ресурсов, либо счётчик Яндекс.Метрики встроен с использованием устаревшего или неподходящего типа загрузки, который конфликтует с политиками безопасности браузера Safari.

---

## Рекомендация

Проверить настройки интеграции Яндекс.Метрики на сайте, убедиться в использовании актуального кода счётчика. При необходимости — настроить сервер для отправки корректных CORS-заголовков.
