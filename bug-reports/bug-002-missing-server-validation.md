## Заголовок
На сайте master164.ru в формах «Получить скидку» и «Запись на бесплатное пробное занятие» отсутствует серверная валидация введённых данных.

---

## Окружение (Environment)

| Компонент | Значение |
|-----------|----------|
| **Устройство** | MacBook Pro 16" 2021 |
| **ОС** | macOS Tahoe 26.2 |
| **Браузеры** | Safari, Яндекс.Браузер |
| **Сервер** | PHP/5.4.45, nginx |
| **Сайт** | [https://master164.ru](https://master164.ru) |
| **Обработчик форм** | [https://master164.ru/form2.php](https://master164.ru/form2.php) |

---

## Предусловие
1. Открыть сайт [https://master164.ru](https://master164.ru) в браузере.
2. Открыть инструменты разработчика (Веб-инспектор) → вкладка **«Сеть» (Network)**.
3. Включить опцию **«Сохранять журнал» (Preserve Log)**.
4. Очистить список запросов.

---

## Шаги воспроизведения

### Для формы 1 («Получить скидку» — в верхней части страницы):
1. Найти форму и ввести некорректные тестовые данные:
   - **Имя:** `ппп`
   - **Телефон:** `700900`
2. Нажать кнопку отправки формы.
3. Во вкладке «Сеть» проанализировать появившийся запрос к `form2.php`.
4. Обратить внимание на сообщение для пользователя.

### Для формы 2 («Запись на бесплатное пробное занятие» — в нижней части страницы):
1. Повторить шаги 1-4, введя данные:
   - **Имя:** `rrrrrr`
   - **Телефон:** `323454`

---

## Фактический результат (Actual Result)

- **Обе формы** успешно отправляют данные на сервер (`/form2.php`).
- **Сервер возвращает** HTTP-статус `200 OK` и тело ответа `true`.
- **Пользователь видит** сообщение: *«Заявка принята! С вами свяжется менеджер»*.
- **Никакой проверки** формата или корректности данных на сервере не происходит.
- **Клиентская валидация** минимальна (только атрибут `required`, стандартные браузерные подсказки).

---

## Данные запросов (Payload)

| Форма | Имя | Телефон | Название формы |
|-------|-----|---------|----------------|
| 1 («Получить скидку») | `ппп` | `700900` | «Заявка на обучение со скидкой» |
| 2 («Пробное занятие») | `rrrrrr` | `323454` | «Заявка на бесплатное пробное обучение» |

---

## Ожидаемый результат (Expected Result)

Хотя письменные требования к форматам полей отсутствуют, серверная валидация должна быть реализована по следующим причинам:
1. **Любые данные от клиента должны проверяться на сервере** как критический уровень безопасности и целостности.
2. **Поле «Телефон»:** Сервер должен проверять, что передана строка, содержащая достаточное количество цифр и похожая на номер телефона.
3. **Поле «Имя»:** Сервер должен проверять, что передана непустая строка разумной длины (например, не менее 2 символов, не содержащая спецсимволов).
4. При несоответствии базовым ожиданиям сервер должен возвращать **ответ с ошибкой**, а не `true`, а пользователь должен видеть информативное сообщение о некорректно заполненном поле.
